# 実装計画 (Python/Tkinter版)

## フェーズ 1: コアロジックの実装

- [ ] **1. プロジェクトのセットアップ**
  - [ ] 1.1. `python_rewrite`内にディレクトリ構造を作成 (`ui/`, `logic/`, `data/`)
  - [ ] 1.2. `requirements.txt`を作成し、依存関係をリストアップ (`Pillow`, `tkinterdnd2`)
  - [ ] 1.3. メインのエントリーポイント `main.py` を作成

- [ ] **2. データモデルと設定管理**
  - [ ] 2.1. `data/config_manager.py` を実装し、`config.json`の読み書きを処理
  - [ ] 2.2. アプリケーション起動時にデフォルト設定ファイルを生成するロジックを実装

- [ ] **3. メタデータ抽出機能**
  - [ ] 3.1. `data/metadata_extractor.py` を実装
  - [ ] 3.2. `os`モジュールを使用してファイルシステム属性（サイズ、更新日時など）を抽出
  - [ ] 3.3. `Pillow`を使用して画像のEXIF情報（撮影日時、カメラモデルなど）を抽出
  - [ ] 3.4. メタデータ抽出に関するユニットテストを作成

- [ ] **4. ルールエンジン**
  - [ ] 4.1. `logic/rule_engine.py` を実装
  - [ ] 4.2. メタデータとルールの条件を照合するロジックを実装
  - [ ] 4.3. 移動先パターンのテンプレート変数（`{year}`など）を展開する機能を実装
  - [ ] 4.4. ルールエンジンのユニットテストを作成

- [ ] **5. ファイル操作**
  - [ ] 5.1. `data/file_operations.py` を実装
  - [ ] 5.2. `os`と`shutil`を使用して、安全なファイルの移動・コピー操作を実装
  - [ ] 5.3. 移動先の親ディレクトリがなければ自動的に作成する機能を実装

- [ ] **6. ファイル処理オーケストレーター**
  - [ ] 6.1. `logic/file_processor.py` を実装
  - [ ] 6.2. ファイルリストを受け取り、メタデータ抽出、ルール適用、ファイル操作を連携させる
  - [ ] 6.3. 処理結果（成功、失敗、エラーメッセージなど）を返すロジックを実装

## フェーズ 2: GUIの実装

- [ ] **7. メインウィンドウ**
  - [ ] 7.1. `ui/main_window.py` を実装
  - [ ] 7.2. `tkinterdnd2`を利用して、ファイルのドラッグ＆ドロップ受け付けエリアを作成
  - [ ] 7.3. 処理中のスピナーや進捗表示などの視覚的フィードバックを実装

- [ ] **8. 設定ウィンドウ**
  - [ ] 8.1. `ui/settings_window.py` を実装
  - [ ] 8.2. ルールの一覧表示、追加、編集、削除を行うUIを作成
  - [ ] 8.3. ルールの優先順位をドラッグ＆ドロップで変更できる機能を実装

- [ ] **9. UIとロジックの統合**
  - [ ] 9.1. D&D時に`FileProcessor`を呼び出すように接続
  - [ ] 9.2. 設定ウィンドウで`ConfigManager`を呼び出して設定の読み込み・保存を行う

- [ ] **10. 結果表示ビュー**
  - [ ] 10.1. `ui/results_view.py` を実装
  - [ ] 10.2. ファイルごとの処理結果（成功/失敗、移動元/先パス）を一覧表示

- [ ] **11. 競合解決ダイアログ**
  - [ ] 11.1. `ui/conflict_dialog.py` を実装
  - [ ] 11.2. ファイル名が競合した場合に、上書き、スキップ、リネームを選択させるダイアログを作成

- [ ] **12. プレビューモード**
  - [ ] 12.1. 実際のファイル操作を行わずに、実行結果をプレビューするロジックを実装
  - [ ] 12.2. プレビュー結果を表示し、ユーザーに実行またはキャンセルを問うUIを実装
