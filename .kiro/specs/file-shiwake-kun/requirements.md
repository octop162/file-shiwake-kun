# 要件定義書

## はじめに

ファイル仕訳け君は、ファイルのメタデータ（画像のEXIF情報、ファイルシステム属性など）に基づいて、ファイルを自動的に整理するクロスプラットフォームGUIアプリケーションです。ユーザーは設定画面にファイルをドラッグ&ドロップするだけで、事前に定義したルールに従ってファイルが適切なディレクトリに移動またはコピーされます。

## 用語集

- **ファイル仕訳け君**: 本システムの名称。ファイルを自動整理するGUIアプリケーション
- **メタデータ**: ファイルに関連する情報。画像の場合はEXIF情報（撮影日時、カメラ機種、GPS位置情報など）、全ファイル共通ではファイルシステム属性（作成日時、更新日時、サイズ、拡張子など）
- **整理ルール**: メタデータに基づいてファイルの移動先を決定するルール
- **設定ファイル**: TOML形式で保存される設定ファイル
- **ドラッグ&ドロップ**: ファイルをマウスでドラッグしてアプリケーションウィンドウにドロップする操作

## 要件

### 要件 1

**ユーザーストーリー:** ユーザーとして、アプリケーションウィンドウにファイルをドラッグ&ドロップしたい。そうすることで、手動のファイル操作なしに複数のファイルを素早く整理できる。

#### 受入基準

1. WHEN ユーザーが1つ以上のファイルをアプリケーションウィンドウ上にドラッグした時 THEN ファイル仕訳け君はドロップゾーンがアクティブであることを示す視覚的フィードバックを表示しなければならない
2. WHEN ユーザーがアプリケーションウィンドウにファイルをドロップした時 THEN ファイル仕訳け君はファイルを受け入れて処理を開始しなければならない
3. WHEN ユーザーがアプリケーションウィンドウにディレクトリをドロップした時 THEN ファイル仕訳け君はそのディレクトリ内の全てのファイルを再帰的に処理しなければならない
4. WHEN 複数のファイルが同時にドロップされた時 THEN ファイル仕訳け君は設定されたルールに従って各ファイルを処理しなければならない
5. WHILE ファイルが処理されている間 THEN ファイル仕訳け君は各ファイルの進捗情報を表示しなければならない

### 要件 2

**ユーザーストーリー:** ユーザーとして、アプリケーションがファイルからメタデータを読み取ってほしい。そうすることで、ファイルの属性に基づいて整理できる。

#### 受入基準

1. WHEN 任意のファイルが処理される時 THEN ファイル仕訳け君は作成日時、更新日時、ファイルサイズ、拡張子を含むファイルシステムメタデータを抽出しなければならない
2. WHEN 画像ファイルが処理される時 THEN ファイル仕訳け君は撮影日時、カメラ機種、GPS座標を含むEXIFメタデータを追加で抽出しなければならない
3. WHEN ファイルに特定のメタデータフィールドが欠けている時 THEN ファイル仕訳け君は失敗せずに欠損データを適切に処理しなければならない
4. WHEN メタデータ抽出が失敗した時 THEN ファイル仕訳け君はエラーをログに記録し、他のファイルの処理を続行しなければならない
5. THE ファイル仕訳け君はEXIF抽出のためにJPEG、PNG、HEIC、RAW形式を含む一般的な画像形式をサポートしなければならない

### 要件 3

**ユーザーストーリー:** ユーザーとして、メタデータに基づいて整理ルールを定義したい。そうすることで、ファイルが自動的に適切なディレクトリに振り分けられる。

#### 受入基準

1. WHEN ユーザーが整理ルールを作成する時 THEN ファイル仕訳け君はマッチさせるメタデータフィールドの指定を許可しなければならない
2. WHEN ユーザーが移動先パスパターンを定義する時 THEN ファイル仕訳け君は `{year}`, `{month}`, `{camera}`, `{extension}` などの変数名を使用したテンプレート形式をサポートしなければならない
3. WHEN ファイルが複数のルールにマッチする時 THEN ファイル仕訳け君は優先順位に従って最初にマッチしたルールを適用しなければならない
4. WHEN ファイルがどのルールにもマッチしない時 THEN ファイル仕訳け君は設定に基づいてファイルをデフォルトディレクトリに移動するか、その場に残さなければならない
5. THE ファイル仕訳け君はファイル拡張子、日付パターン、ファイルサイズ、カメラ機種、GPS位置、カスタムメタデータフィールドに基づくルールをサポートしなければならない

### 要件 4

**ユーザーストーリー:** ユーザーとして、ファイルの移動とコピーを選択したい。そうすることで、必要に応じて元のファイルを保持できる。

#### 受入基準

1. WHEN ユーザーが整理ルールを設定する時 THEN ファイル仕訳け君は移動操作とコピー操作の選択を許可しなければならない
2. WHEN 移動操作が実行される時 THEN ファイル仕訳け君は転送成功後に元の場所からファイルを削除しなければならない
3. WHEN コピー操作が実行される時 THEN ファイル仕訳け君は元の場所に元のファイルを保持しなければならない
4. WHEN ファイル操作が失敗した時 THEN ファイル仕訳け君は元のファイルを削除せず、エラーを報告しなければならない
5. WHEN 移動先にファイルが既に存在する時 THEN ファイル仕訳け君はユーザーに確認を求めなければならない

### 要件 5

**ユーザーストーリー:** ユーザーとして、TOML形式で設定を保存・読み込みたい。そうすることで、整理ルールがセッション間で永続化される。

#### 受入基準

1. WHEN アプリケーションが起動する時 THEN ファイル仕訳け君はアプリケーションディレクトリ内のTOMLファイルから設定を読み込もうと試みなければならない
2. WHEN 設定が変更される時 THEN ファイル仕訳け君はTOMLファイルに変更を保存しなければならない
3. WHEN 設定ファイルが存在しない時 THEN ファイル仕訳け君はデフォルトの設定ファイルを作成しなければならない
4. WHEN 設定ファイルに無効なTOML構文が含まれる時 THEN ファイル仕訳け君はエラーメッセージを表示し、デフォルト設定を使用しなければならない
5. THE ファイル仕訳け君は設定値を検証し、無効なルールやパスを拒否しなければならない

### 要件 6

**ユーザーストーリー:** ユーザーとして、整理ルールを管理する設定画面がほしい。そうすることで、ファイルを手動で編集せずにアプリケーションを簡単に設定できる。

#### 受入基準

1. WHEN アプリケーションが起動する時 THEN ファイル仕訳け君はメインインターフェースとして設定画面を表示しなければならない
2. WHEN ユーザーが新しいルールを追加する時 THEN ファイル仕訳け君はルール条件と移動先パスの入力フィールドを提供しなければならない
3. WHEN ユーザーが既存のルールを変更する時 THEN ファイル仕訳け君は設定を即座に更新しなければならない
4. WHEN ユーザーがルールを削除する時 THEN ファイル仕訳け君は設定からそれを削除し、表示を更新しなければならない
5. WHEN ユーザーがルールを並び替える時 THEN ファイル仕訳け君はルールの優先順位を適切に更新しなければならない

### 要件 7

**ユーザーストーリー:** ユーザーとして、アプリケーションがWindows、macOS、Linuxで動作してほしい。そうすることで、オペレーティングシステムに関係なく使用できる。

#### 受入基準

1. THE ファイル仕訳け君はWindows 10以降のバージョンで動作しなければならない
2. THE ファイル仕訳け君はmacOS 10.15以降のバージョンで動作しなければならない
3. THE ファイル仕訳け君はGTK3をサポートするLinuxディストリビューションで動作しなければならない
4. WHEN ファイルパスが処理される時 THEN ファイル仕訳け君はプラットフォーム固有のパス区切り文字を正しく処理しなければならない
5. WHEN アプリケーションがシステムリソースにアクセスする時 THEN ファイル仕訳け君はクロスプラットフォームなAPIを使用しなければならない

### 要件 8

**ユーザーストーリー:** ユーザーとして、ファイル操作に関する明確なフィードバックがほしい。そうすることで、各ファイルに何が起こったかを知ることができる。

#### 受入基準

1. WHEN ファイルが正常に処理された時 THEN ファイル仕訳け君は元のパスと移動先パスを表示しなければならない
2. WHEN ファイル操作が失敗した時 THEN ファイル仕訳け君は理由を含む明確なエラーメッセージを表示しなければならない
3. WHEN 処理が完了した時 THEN ファイル仕訳け君は成功した操作と失敗した操作のサマリーを表示しなければならない
4. WHILE 操作が進行中の時 THEN ファイル仕訳け君は進捗インジケーターを表示しなければならない
5. THE ファイル仕訳け君はトラブルシューティングのために全てのファイル操作のログを保持しなければならない

### 要件 9

**ユーザーストーリー:** ユーザーとして、ファイルが移動される前に何が起こるかをプレビューしたい。そうすることで、整理ルールが正しく動作することを確認できる。

#### 受入基準

1. WHERE プレビューモードが有効な時 THEN ファイル仕訳け君はファイル操作を実行せずに意図された移動先を表示しなければならない
2. WHEN プレビュー結果が表示される時 THEN ファイル仕訳け君は各ファイルにマッチしたルールを示さなければならない
3. WHEN ユーザーがプレビュー結果を確認する時 THEN ファイル仕訳け君は実際のファイル操作を実行しなければならない
4. WHEN ユーザーがプレビュー後にキャンセルする時 THEN ファイル仕訳け君はファイルを変更せずに操作を破棄しなければならない
5. THE ファイル仕訳け君は設定でプレビューモードの切り替えを許可しなければならない

### 要件 10

**ユーザーストーリー:** ユーザーとして、移動先ディレクトリが自動的に作成されてほしい。そうすることで、フォルダ構造を手動で作成する必要がない。

#### 受入基準

1. WHEN 移動先パスが存在しない時 THEN ファイル仕訳け君は必要な全ての親ディレクトリを作成しなければならない
2. WHEN 権限の問題でディレクトリ作成が失敗した時 THEN ファイル仕訳け君はエラーを報告し、そのファイルをスキップしなければならない
3. WHEN ディレクトリを作成する時 THEN ファイル仕訳け君はオペレーティングシステムに適切な権限を使用しなければならない
4. WHEN 移動先パスに無効な文字が含まれる時 THEN ファイル仕訳け君はパスをサニタイズするか、エラーを報告しなければならない
5. THE ファイル仕訳け君はファイル操作を試みる前に移動先パスを検証しなければならない

### 要件 11

**ユーザーストーリー:** ユーザーとして、ファイル上書き時に確認ダイアログを表示してほしい。そうすることで、重要なファイルを誤って上書きすることを防げる。

#### 受入基準

1. WHEN 移動先に同名のファイルが既に存在する時 THEN ファイル仕訳け君は確認ダイアログを表示しなければならない
2. WHEN 上書き確認ダイアログが表示される時 THEN ファイル仕訳け君は両方のファイルの名前、サイズ、更新日時を表示しなければならない
3. WHEN ユーザーが上書き確認ダイアログで選択する時 THEN ファイル仕訳け君は上書き、スキップ、リネームのオプションを提供しなければならない
4. WHEN ユーザーが「以降も同様に処理」オプションを選択する時 THEN ファイル仕訳け君は同じ条件の競合に対して同じ動作を自動的に適用しなければならない
5. WHEN バッチ処理中に複数の競合が発生する時 THEN ファイル仕訳け君は「すべて上書き」「すべてスキップ」「すべてリネーム」のオプションを提供しなければならない
