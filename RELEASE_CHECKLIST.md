# リリースチェックリスト

このチェックリストは、ファイル仕訳け君の新しいバージョンをリリースする際に使用します。

## リリース前の準備

### 1. コードの品質確認

- [ ] すべてのテストが通ることを確認
  ```bash
  npm run test
  cd src-tauri && cargo test --release
  ```

- [ ] プロパティベーステストが通ることを確認
  ```bash
  cd src-tauri && cargo test --release -- --ignored
  ```

- [ ] Lintエラーがないことを確認
  ```bash
  npm run build
  cd src-tauri && cargo clippy -- -D warnings
  ```

- [ ] 型チェックが通ることを確認
  ```bash
  npx tsc --noEmit
  ```

### 2. バージョン番号の更新

- [ ] `package.json` のバージョンを更新
- [ ] `src-tauri/Cargo.toml` のバージョンを更新
- [ ] `src-tauri/tauri.conf.json` のバージョンを更新
- [ ] バージョン番号が3つのファイルで一致していることを確認

### 3. ドキュメントの更新

- [ ] `CHANGELOG.md` に変更内容を記載
- [ ] `README.md` の更新（必要に応じて）
- [ ] `BUILD.md` の更新（ビルド手順に変更がある場合）

### 4. 動作確認

- [ ] 開発モードで動作確認
  ```bash
  npm run tauri:dev
  ```

- [ ] 主要機能の動作確認
  - [ ] ファイルのドラッグ&ドロップ
  - [ ] メタデータの抽出
  - [ ] ルールの適用
  - [ ] ファイルの移動/コピー
  - [ ] 設定の保存/読み込み
  - [ ] プレビューモード
  - [ ] 競合解決ダイアログ

## ビルドプロセス

### Windows (要件 7.1)

- [ ] Windows 10/11でビルド実行
  ```powershell
  .\scripts\build-all.ps1
  ```

- [ ] 生成されたインストーラーの確認
  - [ ] MSIインストーラーが生成されている
  - [ ] NSISインストーラーが生成されている

- [ ] インストーラーの動作確認
  - [ ] MSIインストーラーでインストール可能
  - [ ] アプリケーションが正常に起動
  - [ ] アンインストールが正常に動作

### macOS (要件 7.2)

- [ ] macOS 10.15以降でビルド実行
  ```bash
  ./scripts/build-all.sh
  ```

- [ ] 生成されたパッケージの確認
  - [ ] Intel版DMGが生成されている
  - [ ] Apple Silicon版DMGが生成されている

- [ ] パッケージの動作確認
  - [ ] DMGをマウント可能
  - [ ] アプリケーションをApplicationsフォルダにコピー可能
  - [ ] アプリケーションが正常に起動
  - [ ] Gatekeeper警告が適切に表示される（署名なしの場合）

### Linux (要件 7.3)

- [ ] GTK3対応Linuxでビルド実行
  ```bash
  ./scripts/build-all.sh
  ```

- [ ] 生成されたパッケージの確認
  - [ ] DEBパッケージが生成されている
  - [ ] AppImageが生成されている

- [ ] パッケージの動作確認
  - [ ] DEBパッケージでインストール可能（Ubuntu/Debian）
  - [ ] AppImageが実行可能
  - [ ] アプリケーションが正常に起動

## 緊急時の対応

### ロールバック手順

1. 問題のあるリリースをGitHubでDraftに変更
2. 前のバージョンのダウンロードリンクを案内
3. 問題を修正した新しいバージョンをリリース

### ホットフィックス手順

1. `hotfix/v0.1.1` ブランチを作成
2. 問題を修正
3. テストを実行
4. マージしてリリース
